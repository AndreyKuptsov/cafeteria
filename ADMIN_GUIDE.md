# 🔐 Руководство по админ-панели Cafeteria.ru

## Доступ к админ-панели

### URL
```
http://localhost:3001/admin/login
```

### Тестовые учетные данные
```
Email: admin@cafeteria.ru
Пароль: admin123
```

## Создание нового администратора

Для создания нового администратора выполните:

```bash
cd server
npm run create-admin
```

Скрипт создаст пользователя с ролью `admin` и выведет учетные данные.

## Возможности админ-панели

### 📋 Управление меню
- **Просмотр** всех блюд с фильтрацией
- **Добавление** новых блюд с полями:
  - Название (RU/EN)
  - Описание (RU/EN)
  - Категория (завтраки, десерты, салаты, супы, паста, основные блюда, напитки)
  - Цена
  - URL изображения
  - Аллергены
  - Статус доступности
  - Рекомендуемое блюдо
- **Редактирование** существующих блюд
- **Удаление** блюд

### 🛒 Управление заказами
- Просмотр всех заказов
- Информация о клиенте (имя, телефон)
- Сумма заказа
- Статус заказа (pending, confirmed, completed, cancelled)

### 📅 Управление бронированиями
- Просмотр всех бронирований столов
- Информация о клиенте
- Дата и время бронирования
- Количество гостей
- Статус бронирования (pending, confirmed, cancelled)

## Роли пользователей

### Admin (Администратор)
- Полный доступ ко всем функциям
- Управление меню (создание, редактирование, удаление)
- Просмотр заказов и бронирований

### Manager (Менеджер)
- Просмотр заказов и бронирований
- Управление меню (создание, редактирование, удаление)

### Customer (Клиент)
- Обычный пользователь сайта
- Может делать заказы и бронирования

## API Endpoints для админа

### Аутентификация
```
POST /api/auth/login
POST /api/auth/register
GET  /api/auth/me
PUT  /api/auth/profile
PUT  /api/auth/change-password
```

### Меню (требуется admin роль)
```
POST   /api/menu          - Создать блюдо
PUT    /api/menu/:id      - Обновить блюдо
DELETE /api/menu/:id      - Удалить блюдо
```

### Публичные endpoints (без авторизации)
```
GET /api/menu             - Получить все блюда
GET /api/menu/:id         - Получить блюдо по ID
GET /api/orders           - Получить заказы
GET /api/bookings         - Получить бронирования
```

## Безопасность

### JWT Токены
- Токены хранятся в localStorage
- Автоматически добавляются к каждому запросу через axios interceptor
- Срок действия: 7 дней

### Защита маршрутов
- Все админские операции (POST, PUT, DELETE) защищены middleware `authenticateToken` и `requireAdmin`
- Проверка роли пользователя на сервере
- Автоматический редирект на страницу логина при отсутствии доступа

### Хеширование паролей
- Используется bcrypt с salt rounds = 10
- Пароли никогда не хранятся в открытом виде

## Разработка

### Добавление новых админских функций

1. **Backend**: Создайте защищенный маршрут
```javascript
router.post('/api/resource', authenticateToken, requireAdmin, async (req, res) => {
  // ваш код
});
```

2. **Frontend**: Используйте useAuth hook
```javascript
import { useAuth } from '../../context/AuthContext';

function MyComponent() {
  const { user, isAdmin } = useAuth();
  
  if (!isAdmin()) {
    return <div>Access denied</div>;
  }
  
  // ваш компонент
}
```

## Troubleshooting

### Не могу войти в админку
1. Проверьте, что сервер запущен: `npm run dev` в корне проекта
2. Убедитесь, что админ создан: `cd server && npm run create-admin`
3. Проверьте консоль браузера на ошибки

### Ошибка "Invalid token"
1. Очистите localStorage: `localStorage.clear()` в консоли браузера
2. Войдите заново

### Не отображаются данные
1. Проверьте, что API сервер работает на порту 5000
2. Откройте Network tab в DevTools и проверьте запросы
3. Убедитесь, что CORS настроен правильно
